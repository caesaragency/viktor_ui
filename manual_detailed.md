## Обзор проекта

**Viktor UI** - это система шаблонов веб-сайтов для ресторанов, основанная на Next.js, предназначенная для создания красивых, настраиваемых веб-сайтов ресторанов. Проект предоставляет модульную архитектуру, где каждая секция веб-сайта (Hero, About, Menu, Hours, Gallery, Contact) реализована как переиспользуемый React компонент.

### Ключевые особенности
- **Многостраничная архитектура**: Отдельные страницы для Главной, Меню, Часов работы, Галереи и Контактов
- **Система тем**: Динамическая тематизация с CSS переменными, поддерживающая светлый/темный режимы
- **Интернационализация**: Встроенная поддержка нескольких языков (английский/испанский)
- **Адаптивный дизайн**: Подход mobile-first с Tailwind CSS
- **Типобезопасность**: Полная реализация TypeScript
- **Компонентный подход**: Модульные компоненты секций для легкой кастомизации

## Основные компоненты

### 1. Корневой Layout (`app/layout.tsx`)

Корневой layout предоставляет HTML структуру и оборачивает все страницы компонентом `RestaurantLayout`.

**Ключевые особенности:**
- Устанавливает атрибут lang для HTML
- Предоставляет обертку body
- Минимальная структура (фактический layout в RestaurantLayout)

**Экспорты:**
- `default RootLayout`: Обязательный экспорт по умолчанию для Next.js

### 2. Restaurant Layout (`app/layout.tsx` - RestaurantLayout)

Основной компонент layout, который обеспечивает единообразную структуру на всех страницах.

**Особенности:**
- **Липкий заголовок**: Навигационная панель, остающаяся вверху при прокрутке
- **Система тем**: Динамически применяет CSS переменные на основе темы ресторана
- **Адаптивная навигация**: Мобильное меню
- **Paper контейнер**: Область контента с закругленными границами и тенью
- **Фон кирпичной стены**: Опциональный текстурированный паттерн фона

**Пропсы:**
```typescript
{
  restaurant: RestaurantRow;
  children: ReactNode;
}
```

**Устанавливаемые переменные темы:**
- `--brand-color`: Основной цвет бренда
- `--accent-color`: Акцентный/вторичный цвет
- `--page-bg`: Фон страницы
- `--paper-bg`: Фон области контента
- `--body-text`: Основной цвет текста
- `--secondary-text`: Вторичный цвет текста

**Элементы навигации:**
- Главная (`/`)
- Меню (`/menu`)
- Часы работы (`/hours`)
- Резервирование (`/contact`)

**Адаптивное поведение:**
- Заголовок адаптирует отступы на разных размерах экрана
- Текст навигации масштабируется соответствующе
- Область контента имеет адаптивные отступы

## Структура данных

### Определения типов (`app/types.ts`)

#### LocalizedText
```typescript
type LocalizedText = Record<string, string>;
```
Используется для мультиязычного контента. Ключи - коды языков (например, "en", "es"), значения - переведенные строки.

#### RestaurantInfoLocation
```typescript
interface RestaurantInfoLocation {
  address?: string;
  city?: string;
  country?: string;
}
```

#### RestaurantInfoContact
```typescript
interface RestaurantInfoContact {
  phone?: string;
  email?: string;
}
```

#### RestaurantInfo
```typescript
interface RestaurantInfo {
  tagline?: LocalizedText;
  description?: LocalizedText;
  about_text?: LocalizedText;  // Расширенный контент о нас
  location?: RestaurantInfoLocation;
  contact?: RestaurantInfoContact;
  hero_image_path?: string;
  logo_path?: string;
  [key: string]: unknown;  // Позволяет дополнительные поля
}
```

#### RestaurantTheme
```typescript
interface RestaurantTheme {
  mode?: 'light' | 'dark' | string;
  brand?: string;              // Цвет бренда (hex)
  accent?: string;             // Акцентный цвет (hex)
  background?: string;          // Устаревший фон
  pageBackground?: string;      // Цвет фона страницы
  paperBackground?: string;     // Фон области контента
  mutedText?: string;          // Приглушенный цвет текста
  brick?: boolean;             // Включить паттерн кирпичной стены
  brickLine?: string;          // Цвет линий кирпича
  [key: string]: unknown;
}
```

#### RestaurantRow
```typescript
interface RestaurantRow {
  id: string;
  slug: string;
  name: string;
  is_published: boolean;
  deleted_at: string | null;
  theme: RestaurantTheme | null;
  info: RestaurantInfo | null;
  settings: Record<string, unknown> | null;
}
```

#### MenuCategory
```typescript
interface MenuCategory {
  id: string;
  restaurant_id: string;
  name: LocalizedText;
  description: LocalizedText | null;
  slug: string;
  sort_order: number | null;
  is_available: boolean | null;
}
```

#### MenuItem
```typescript
interface MenuItem {
  id: string;
  restaurant_id: string;
  category_id: string | null;
  name: LocalizedText;
  description: LocalizedText | null;
  price: number | string;
  images: string[] | null;
  tags: string[] | null;
  is_available: boolean | null;
  sort_order: number | null;
}
```

#### OpeningHour
```typescript
interface OpeningHour {
  id: string;
  restaurant_id: string;
  weekday: number;        // 0-6 (Пн-Вс)
  opens_at: string;       // Формат "HH:MM:SS"
  closes_at: string;      // Формат "HH:MM:SS"
  is_closed: boolean;
}
```

## Система стилей

### Глобальные стили (`app/globals.css`)

#### CSS переменные
```css
:root {
  --background: #ffffff;
  --foreground: #171717;
  --brand-color: #b22222;
  --accent-color: #f5deb3;
  --body-text: #1f2937;
  --secondary-text: #475569;
  --page-bg: #faf9f3;
  --paper-bg: #ffffff;
}
```

**Примечание:** Переменные темы ресторана динамически переопределяют эти значения по умолчанию через инлайн стили.

#### Паттерн кирпичной стены
Класс `.brick-wall` создает тонкий текстурированный фон используя CSS градиенты:
- Горизонтальные линии каждые 2px
- Вертикальные линии каждые 40px
- Создает эффект кирпичного паттерна

#### Поддержка темной темы
Media query для `prefers-color-scheme: dark` настраивает базовые цвета.

### Tailwind CSS

Проект использует Tailwind CSS 4 с:
- Утилитарным подходом
- Адаптивными брейкпоинтами (sm, md, lg)
- Интеграцией пользовательских цветов через CSS переменные
- Обработкой PostCSS

### Применение темы

Цвета темы применяются через:
1. **CSS переменные**: Устанавливаются в rootStyle `RestaurantLayout`
2. **Инлайн стили**: Используются для динамических цветов
3. **Классы Tailwind**: Комбинируются с CSS переменными

**Пример:**
```tsx
style={{ color: 'var(--brand-color)' } as CSSProperties}
```

## Маршрутизация страниц

### Файловая маршрутизация

Next.js App Router использует файловую структуру для маршрутизации:

| Путь файла | URL маршрут | Описание |
|-----------|-------------|----------|
| `app/page.tsx` | `/` | Главная страница |
| `app/menu/page.tsx` | `/menu` | Страница меню |
| `app/hours/page.tsx` | `/hours` | Страница часов работы |
| `app/gallery/page.tsx` | `/gallery` | Страница галереи |
| `app/contact/page.tsx` | `/contact` | Страница контактов |

### Структура страницы

Каждая страница следует этому паттерну:
1. Импорт компонентов секций
2. Импорт `RestaurantLayout`
3. Импорт и приведение типов примеров данных
4. Рендер `RestaurantLayout` с компонентами секций

**Пример (Страница меню):**
```typescript
export default function MenuPage() {
  const restaurant = sampleData.restaurant as RestaurantRow;
  const categories = sampleData.categories as MenuCategory[];
  const items = sampleData.items as MenuItem[];
  const lang = 'en';

  return (
    <RestaurantLayout restaurant={restaurant}>
      <MenuVariant1 categories={categories} items={items} lang={lang} />
      <FooterVariant1 restaurant={restaurant} />
    </RestaurantLayout>
  );
}
```

## Вспомогательные функции

### `getLocalizedText` (`sections/helpers.ts`)

**Назначение:** Извлекает локализованный текст из объекта LocalizedText.

**Сигнатура:**
```typescript
function getLocalizedText(
  obj: Record<string, string> | undefined | null,
  lang: string
): string
```

**Логика:**
1. Возвращает пустую строку если obj null/undefined
2. Возвращает `obj[lang]` если доступно
3. Откат к `obj.en` если lang не найден
4. Откат к первому доступному значению
5. Возвращает пустую строку если нет значений

**Использование:**
```typescript
const tagline = getLocalizedText(restaurant.info?.tagline, 'en');
```

### `normalizeImagePath` (`sections/helpers.ts`)

**Назначение:** Нормализует пути изображений для единообразного использования.

**Сигнатура:**
```typescript
function normalizeImagePath(path: string | undefined | null): string | null
```

**Логика:**
1. Возвращает null если path null/undefined
2. Возвращает path как есть если это полный URL (http:// или https://)
3. Обеспечивает начало пути с `/` если относительный
4. Возвращает null если path пустой

**Использование:**
```typescript
const imagePath = normalizeImagePath(restaurant.info?.hero_image_path);
```

### `getInitials` (`sections/helpers.ts`)

**Назначение:** Извлекает инициалы из названия ресторана.

**Сигнатура:**
```typescript
function getInitials(name: string): string
```

**Логика:**
1. Разделяет имя по пробелам
2. Берет первую букву каждого слова
3. Преобразует в верхний регистр
4. Возвращает первые 2 символа

**Использование:**
```typescript
const initials = getInitials("Caesar Trattoria"); // "CT"
```

## Продвинутые темы

### CSS переменные vs инлайн стили

**CSS переменные:**
- Устанавливаются один раз в `RestaurantLayout`
- Доступны во всем дереве компонентов
- Могут быть переопределены для каждого компонента

**Инлайн стили:**
- Используются для динамических значений
- Типобезопасны с TypeScript
- Переопределяют CSS переменные при необходимости

### Варианты компонентов

Текущее соглашение именования: `[Component]Variant1`

**Будущее расширение:**
- Создание `HeroVariant2`, `MenuVariant2` и т.д.
- Переключение вариантов через пропсы или конфигурацию
- Поддержание единообразного API

### Загрузка данных

**Текущее:**
- Статический импорт JSON
- Требуется приведение типов
- Нет серверной выборки

**Будущие опции:**
- API роуты для динамических данных
- Интеграция с базой данных
- Серверная выборка данных

### Оптимизация производительности

**Текущие оптимизации:**
- Компонент Next.js Image
- Разделение кода (автоматическое)
- Статическая генерация (где возможно)

**Дополнительные опции:**
- Ленивая загрузка изображений
- Ленивая загрузка компонентов
- Статическая генерация сайта
- Инкрементальная статическая регенерация
